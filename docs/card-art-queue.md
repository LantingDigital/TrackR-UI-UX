## Cedar Point

### Steel Vengeance
- **Status:** HAS ART (`steel-vengeance`)
- **Park:** Cedar Point, Sandusky OH
- **Type:** RMC Hybrid (wood structure, steel IBox track)
- **Best source images:**
  - RCDB gallery (200+ community photos): https://rcdb.com/15411.htm
  - Coasterpedia category (27 files, many dramatic angles): https://coasterpedia.net/wiki/Category:Steel_Vengeance
  - CoasterBuzz featured photo: https://coasterbuzz.com/RollerCoasters/Photo/7810
  - Flickr multi-exposure shot (dramatic): https://www.flickr.com/photos/davidfmorton/32275004698
  - Theme Park James gallery: https://www.themeparkjames.co.uk/theme-parks/northamerica/usa/cedar-point/rides/steel-vengeance/
- **Best angle notes:** Low angle looking up at the 90-degree first drop. The twisted wood structure underneath makes for an incredible foreground. Portrait crops well from below the initial descent. Avoid snow shots (Ohio = seasonal).

## NanoBanana Workflow Checklist (per coaster)

- [ ] Find source image (use URLs above -- RCDB galleries are the most reliable)
- [ ] Pick portrait-friendly crop with dramatic angle and open sky
- [ ] Open Gemini (personal or business Google account)
- [ ] Paste the NanoBanana prompt from the aspiration file
- [ ] Add coaster-type note to prompt (sit-down, inverted, 4D, hybrid, etc.)
- [ ] Add weather note if source has snow: "remove snow, make it look like spring/summer"
- [ ] Add train note if source has no train: "do NOT add a train to the track"
- [ ] Upload source image + submit
- [ ] Wait for NanoBanana 2 result
- [ ] Click three dots under image -> "Redo with Pro"
- [ ] Evaluate against quality criteria (see aspiration file)
- [ ] If not right: hit redo (up to ~5 attempts before switching source image)
- [ ] Save final image, remove watermark if present
- [ ] Rename to coaster ID format (e.g., `iron-gwazi.webp`)
- [ ] Place in `assets/cards/`
- [ ] Add entry to `src/data/cardArt.ts`
- [ ] Hot reload and verify in-app appearance
- [ ] Adjust `CARD_ART_FOCAL` if the default center crop is off

## Notes on Source Image Quality

**Best sources in order of reliability:**
1. RCDB.com (community photos, many dramatic angles, free to browse)
2. Wikipedia article lead images (usually high-quality, well-composed)
3. Official park media pages (Universal, Busch Gardens -- high-res, clean)
4. CoasterBuzz featured photos (curated, high quality)
5. Coasterpedia (solid but smaller collection)
6. Flickr coaster enthusiast accounts (hit or miss but occasionally exceptional)

**Avoid:**
- Stock photo sites (Getty, iStock) -- watermarked and the angles are often generic press shots
- TikTok/Instagram -- too compressed
- Onride photos -- wrong perspective for card art

**Ideal source image characteristics for NanoBanana:**
- Clear sky in background (gradient sky is the NanoBanana default)
- Low angle looking up at the structure
- Train on the track (unless the coaster ID asks for no train)
- No people visible in foreground (or easily croppable)
- No heavy park signage in frame
- Portrait-friendly composition or croppable to 2:3
