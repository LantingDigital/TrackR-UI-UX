# NanoBanana = Internal Dev Terminology ONLY

## The Rule

"NanoBanana" is the name of the AI image generation model (Google Gemini's image tool) used to create TrackR's coaster card art. It is an INTERNAL development term.

**NEVER use "NanoBanana" in any user-facing context.** This includes:
- App UI (screens, buttons, labels, descriptions)
- User-facing documentation
- Marketing materials
- Community posts
- Emails
- App Store listing

## What to Call It Instead

- **"Coaster art"** — when referring to the artwork itself
- **"Cards"** — when referring to the physical or digital trading cards
- **"Card art"** — when referring to the art on the cards
- **"TrackR cards"** — when referring to the product/brand

## Where NanoBanana IS Acceptable

- Agent definitions (`.claude/agents/`)
- Developer documentation (`.claude/` docs, internal docs)
- Pipeline code and scripts (`assets/card-art-pipeline/`)
- Git commit messages referencing the generation process
- Internal team communication

## Why

NanoBanana is a Google product name that TrackR has no affiliation with. Using it publicly would:
1. Confuse users who don't know what it means
2. Associate TrackR's brand with a third-party AI tool
3. Potentially create trademark/branding issues
